# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

**ClipChat** — chat-first video editing platform. Natural language → FFmpeg operations.

Two packages in this monorepo:
- **`packages/engine`** — open-source backend: Express API, BullMQ job queue, FFmpeg worker, AI chat (OpenRouter), MCP server (stdio)
- **`packages/web`** — self-hosted Next.js UI: chat interface, video player, edit timeline, session sidebar

## Commands

### Engine (packages/engine)

```bash
npm install                                    # install all workspaces
npm run dev -w packages/engine                 # start dev server on :3000 (tsx watch)
npm run build -w packages/engine               # compile TypeScript
npm test -w packages/engine                    # run all tests (Vitest)
npm test -w packages/engine -- test/ffmpeg/trim.test.ts  # run single test file
npm run db:generate -w packages/engine         # generate Drizzle migrations
npm run db:migrate -w packages/engine          # apply migrations
npm run create-api-key -w packages/engine -- "label"     # generate an API key
```

### Web UI (packages/web)

```bash
npm run dev:web                                # start Next.js dev server on :3001
npm run build -w packages/web                  # build for production
```

### Docker (full stack)

```bash
ENGINE_API_KEY=$KEY docker compose up -d       # start all 4 services
docker compose up -d postgres redis            # infra only (for local engine dev)
docker compose down
```

### Environment

Copy `.env.example` → `.env`. Key vars:
- `DATABASE_URL` — PostgreSQL connection string
- `REDIS_URL` — Redis (default: `redis://localhost:6379`)
- `STORAGE_DRIVER` — `local` (default) or `s3`
- `OPENROUTER_API_KEY` — required for `/api/v1/chat`
- `ENGINE_URL` — engine base URL for web proxy (default: `http://localhost:3000`)
- `ENGINE_API_KEY` — Bearer token used by web proxy to call the engine (generate with `create-api-key`)

## Engine Architecture

```
packages/engine/src/
├── index.ts            # Entry: starts HTTP server + worker; --mcp flag enables MCP server
├── api/
│   ├── index.ts        # Express app factory (CORS, routes, requireApiKey middleware)
│   ├── middleware/      # auth.ts (SHA-256 Bearer → apiKeyId), errors.ts
│   └── routes/         # health, files, jobs, tools, chat, sessions
├── ffmpeg/             # One file per tool: executor.ts + info/trim/merge/resize/audio/text/speed/export
├── queue/              # BullMQ setup (index.ts) and worker (worker.ts — TOOL_MAP dispatch)
├── ai/
│   └── tools.ts        # buildOpenRouterTools(): Zod → JSON Schema; createOpenRouterClient()
├── services/
│   └── session.ts      # createSession, saveUserMessage, saveAssistantMessage, saveToolMessage, buildMessageHistory
├── storage/            # StorageAdapter interface; LocalStorageAdapter and S3StorageAdapter
├── db/
│   ├── schema.ts       # Drizzle tables: files, jobs, api_keys, sessions, chat_messages
│   └── migrations/     # Generated by drizzle-kit
├── types/              # Zod schemas for all 10 tools + Job, FileRecord, AppError, Session, ChatMessage
└── mcp/                # McpServer with StdioServerTransport; calls FFmpeg directly (no queue)
```

**Job flow (async):** `POST /api/v1/jobs` → BullMQ → worker dispatches to `ffmpeg/*.ts` → updates DB progress/status → client polls `GET /jobs/:id` or streams via SSE `GET /jobs/:id/stream` (500ms interval).

**Chat flow:** `POST /api/v1/chat { session_id?, message, file_id? }` → resolves file path → OpenRouter streaming with tool calling → SSE events: `text` (delta), `tool_call` (assembled), `done` (with session_id). Tool calls submit jobs to BullMQ; all turns persisted to `sessions`/`chat_messages`.

**Auth:** Bearer token → SHA-256 → lookup in `api_keys`. `req.apiKeyId` available in all protected routes. Generate keys with `create-api-key` script.

**Storage:** `createStorage()` returns `LocalStorageAdapter` or `S3StorageAdapter` based on `STORAGE_DRIVER`. Files persisted to DB on upload so they survive restarts. `GET /files/:id/content` streams raw bytes (supports video playback with `Accept-Ranges`).

**MCP:** `node dist/index.js --mcp` exposes all 10 tools via stdio transport, calling FFmpeg synchronously (no queue) for agent workflows.

## Web Architecture

```
packages/web/
├── app/
│   ├── layout.tsx                  # Root layout: QueryProvider, Sidebar, dark mode
│   ├── chat/page.tsx               # /chat — new session
│   ├── chat/[sessionId]/page.tsx   # existing session (async params pattern)
│   └── api/                        # Proxy route handlers (ENGINE_API_KEY injected server-side)
│       ├── chat/route.ts           # SSE pass-through: new Response(upstream.body)
│       ├── files/upload/route.ts   # Multipart proxy with duplex: 'half'
│       ├── files/[id]/route.ts
│       ├── files/[id]/content/route.ts
│       ├── jobs/[id]/route.ts
│       └── sessions/[id]/messages/route.ts
├── components/
│   ├── chat/                       # chat-view, message-list, message-bubble, tool-call-card,
│   │                               #   video-player, edit-timeline, timeline-step, chat-input
│   ├── layout/                     # sidebar, sidebar-item
│   ├── providers/query-provider.tsx
│   └── ui/                         # shadcn/ui: button, card, badge, progress, scroll-area, etc.
├── hooks/
│   ├── use-chat.ts                 # SSE streaming; reconstructs session history on mount
│   ├── use-job.ts                  # react-query polling (stops at completed/failed)
│   └── use-sessions-list.ts       # react-query, 10s refetch
└── lib/
    ├── engine-client.ts            # Types (FileRecord, Job, Session, ChatMessage) + fetch helpers
    └── utils.ts                    # cn() helper
```

**Proxy pattern:** All `app/api/` routes forward to the engine with `Authorization: Bearer ${ENGINE_API_KEY}`. The key never reaches the browser. SSE streams pass through as `new Response(upstream.body)` with `X-Accel-Buffering: no`. Multipart uploads use `duplex: 'half'` for streaming body forwarding.

**`use-chat.ts`:** On mount with `initialSessionId`, fetches DB messages and reconstructs the display array (skipping `role='tool'` rows — internal to OpenRouter history). Sends to `/api/chat`, reads `ReadableStream`, parses SSE lines, handles `text`/`tool_call`/`done`/`error` events. Routes to `/chat/[sessionId]` after first message.

**`packages/web` does not import from `packages/engine`** — types in `engine-client.ts` are manually mirrored.

## Testing

Engine tests require running Postgres and Redis:

```bash
docker compose up -d postgres redis
export DATABASE_URL=postgres://postgres:clipchat@localhost:5432/clipchat
export REDIS_URL=redis://localhost:6379
npm test -w packages/engine
```

FFmpeg tests generate synthetic fixtures on first run (`test/helpers/fixtures.ts`). Fixtures are gitignored.

`singleFork: true` in vitest config prevents parallel FFmpeg processes from conflicting. Test timeout is 30s.
